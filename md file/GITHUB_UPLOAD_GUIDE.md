# GitHub へのアップロード完全ガイド

## 📋 前提条件

以下がインストールされていることを確認してください：

```bash
# Git のバージョン確認
git --version

# GitHub アカウントの確認
# https://github.com にアクセスしてログイン
```

---

## 🚀 ステップバイステップ手順

### ステップ 1️⃣: GitHub で新しいリポジトリを作成

#### 1.1 GitHub にログイン
```
https://github.com にアクセス
右上のプロフィールアイコン → "Your repositories"
```

#### 1.2 新しいリポジトリを作成
```
1. "New" ボタンをクリック
2. Repository name: pdf-workflow-manager
3. Description: Enterprise-grade PDF processing system
4. Public/Private を選択（Public 推奨）
5. README.md にチェック（後で上書き）
6. .gitignore を選択 → "Python"
7. License を選択 → "MIT" または "Apache 2.0"
8. "Create repository" をクリック
```

#### 1.3 作成後の画面から リモート URL をコピー
```
例：https://github.com/your-username/pdf-workflow-manager.git
または ssh://git@github.com:your-username/pdf-workflow-manager.git
```

---

### ステップ 2️⃣: ローカルで Git を初期化

#### 2.1 プロジェクトディレクトリに移動
```bash
cd /path/to/pdf-workflow-manager
```

#### 2.2 Git を初期化（GitHub から README.md をダウンロードしていない場合）
```bash
git init
```

#### 2.3 リモートリポジトリを登録
```bash
# HTTPS の場合
git remote add origin https://github.com/your-username/pdf-workflow-manager.git

# SSH の場合（推奨）
git remote add origin git@github.com:your-username/pdf-workflow-manager.git
```

#### 2.4 確認
```bash
git remote -v
# 出力例:
# origin  https://github.com/your-username/pdf-workflow-manager.git (fetch)
# origin  https://github.com/your-username/pdf-workflow-manager.git (push)
```

### ステップ 6️⃣: GitHub にプッシュ

## 🔐 SSH キー設定（オプション・推奨）

毎回パスワードを入力しなくて済むようにします。

### SSH キーの生成

```bash
# Ed25519 キー（推奨）
ssh-keygen -t ed25519 -C "your-email@example.com"

# または RSA キー（互換性重視）
ssh-keygen -t rsa -b 4096 -C "your-email@example.com"
```

### SSH キーを GitHub に登録

```bash
# 公開鍵をコピー（macOS）
cat ~/.ssh/id_ed25519.pub | pbcopy

# または Windows PowerShell
Get-Content ~/.ssh/id_ed25519.pub | Set-Clipboard

# または Linux
cat ~/.ssh/id_ed25519.pub | xclip -selection clipboard
```

**GitHub で登録：**
1. GitHub → Settings → SSH and GPG keys
2. "New SSH key"
3. Title: "My Computer" など
4. Key にペースト
5. "Add SSH key" をクリック

## 🐛 トラブルシューティング

### エラー: "fatal: remote origin already exists"

```bash
# 既存のリモートを削除
git remote remove origin

# 再度追加
git remote add origin https://github.com/your-username/pdf-workflow-manager.git
```

### エラー: "permission denied (publickey)"

```bash
# SSH キーが正しく登録されているか確認
ssh -T git@github.com
# 出力例: Hi username! You've successfully authenticated...
```

### エラー: "Updates were rejected because the tip of your current branch is behind"

```bash
# GitHub から最新を引っ張る
git pull origin main

# 競合がない場合はそのままプッシュ可能
git push origin main
```

### 初回 GitHub から README.md をダウンロードした場合

```bash
# 以下の順序で実行
git pull origin main --allow-unrelated-histories
git add .
git commit -m "Initial commit"
git push origin main
```
